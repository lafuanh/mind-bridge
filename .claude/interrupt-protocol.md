# Interrupt Protocol

## The Promise

When Noupal writes `#interrupt`, he is in flow state and chose NOT to break it.
He trusts the system to hold the thought until 22:00.

**Never break this promise.**

## Rules

```
05:00 and 11:45 syncs:
  → HOLD all #interrupt entries
  → Do NOT classify, route, or create anything from them
  → Do NOT strikethrough
  → Note count in Master Review: "N interrupts held for 22:00"

22:00 sync only:
  → Process by content type (see classify-route.md)
  → Add "(was #interrupt, held from [original time])" at destination
  → Strikethrough in dump with timestamp
```

## Why

Flow state is finite and rare. Interrupting it for something non-urgent costs more than the task is worth.
The interrupt buffer is the OS honoring the user's judgment about their own focus.
