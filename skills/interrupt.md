# Skill: Interrupt Protocol
> Sacred. Do not break this.

---

## WHEN

An entry is tagged `#interrupt` — meaning Noupal captured it while in flow state
and explicitly chose NOT to break focus to deal with it.

The `#interrupt` tag is a promise to the user:
> "I am trusting the system to hold this. I will not lose it. But I will not let it break my focus."

---

## DO

### At 05:00 (morning brief) and 11:45 (midday flush):

```
1. Identify all #interrupt entries in /dump
2. DO NOT classify them
3. DO NOT route them
4. DO NOT create any tasks, events, or notes from them
5. DO NOT strikethrough them
6. Count them: note "N interrupts held for 22:00" in Master Review
7. Leave them exactly as they are in the dump
```

### At 22:00 (night digest):

```
1. Read ALL held #interrupt entries
2. Process each one by its actual content type:
   - #interrupt + task content → route as #todo
   - #interrupt + schedule content → route as #sched
   - #interrupt + knowledge content → route as #know
   - #interrupt + feeling → route as #feel
   etc.
3. Add "Was #interrupt" note at each destination:
   - In Notion: append "(was #interrupt, held from [time])"
   - In Obsidian: add frontmatter: interrupt: true
4. Strikethrough at 22:00 with timestamp
```

---

## WRITE TO

At 05:00 / 11:45: **nowhere** — hold only.

At 22:00: same destinations as the entry's content type (see `classify-route.md`).

---

## WHY THIS MATTERS

Flow state is one of the most valuable cognitive states a person can be in.
Breaking it for a task that can wait is a tax on deep work.

The interrupt buffer exists so that capturing a thought does not cost focus.
When Noupal writes `#interrupt`, he is saying: "I trust you to hold this."
Honor that trust. Never process it early.
